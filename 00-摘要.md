门诊业务涉及科室、医生、患者、预约、排班、就诊记录与处方等多个环节，信息在不同岗位之间高频流转，现有依赖纸质登记或分散系统的管理方式容易造成数据口径不一致、检索效率低、流程割裂与协同滞后，跨科室业务衔接难以形成闭环，权限与数据标准缺失使预约、诊疗与用药的连续管理以及统计分析和质量追踪受到制约。本文面向医疗机构门诊流程，设计并实现前后端分离的医疗信息系统，前端采用 Vue 3、Vite 与 Element Plus 构建业务界面，后端基于 Spring Boot 3.2、MyBatis-Plus、Spring Security、JWT 与 MySQL 提供统一服务层，覆盖用户与权限、科室、医生、患者、预约、排班、就诊记录、药品及处方等核心模块，并围绕预约申请、排班维护、就诊记录录入、处方开具与药品流转形成闭环，通过统一数据模型与接口规范提升信息一致性、可追溯性与维护效率，支撑门诊服务效率提升、用药安全控制、统计报表与管理决策。关键词：医疗信息系统；门诊管理；处方管理；Spring Boot
