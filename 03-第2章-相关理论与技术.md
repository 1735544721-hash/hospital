第 2 章 相关理论与技术

2.1 Spring Boot 框架介绍

Spring Boot 是 Spring 技术栈的轻量化封装，强调约定优于配置，通过自动配置与 starter 机制快速集成 Web 与数据访问能力，减少传统 XML 配置带来的开发成本。本系统后端基于 Spring Boot 3.2 与 Java 17 构建，启用 spring-boot-starter-web 提供 REST 接口，结合 spring-boot-starter-jdbc 与 spring-boot-starter-data-jdbc 保证基础数据访问能力，并通过 mybatis-plus-spring-boot3-starter 完成 ORM 映射与分页查询；权限层引入 Spring Security 与 JWT 形成登录鉴权链路，邮件与通知能力由 spring-boot-starter-mail 提供，接口文档借助 Knife4j 输出，便于前后端联调。系统采用 Controller、Service、Mapper 分层组织业务逻辑，并结合 Spring AOP 统一日志与异常处理，使预约、排班、诊疗记录与处方管理等流程具备清晰的职责边界与可维护性。

2.2 Vue.js 前端框架

Vue.js 是渐进式 JavaScript 前端框架，核心特征包括组件化开发、响应式数据驱动与单文件组件，适合构建单页面应用并提升界面复用效率。

本系统前端基于 Vue 3 与 Vite 4 构建，使用 Vue Router 组织预约、排班、科室、医生、患者、处方与药品等页面路由，通过 Pinia 与 Vuex 维护登录态与查询条件，结合 Axios 统一封装接口请求；Element Plus 与图标库提供表格、表单与弹窗组件，提高数据录入与查询的交互一致性，配合 Sass 维护主题样式，使前端页面在高频业务操作场景下保持清晰的视图层结构与稳定的响应体验。

2.3 MyBatis-Plus 数据访问框架

MyBatis-Plus 是对 MyBatis 的增强型数据访问框架，在保留原生 SQL 灵活性的同时提供通用 CRUD、条件构造器与分页能力，能够减少大量重复映射代码并提升数据层开发效率。其轻量化设计适合与 Spring Boot 进行整合，通过约定式的 Mapper 接口即可完成实体持久化管理，既满足医疗信息系统对数据一致性的要求，也便于在后续扩展复杂查询时保持可控的 SQL 逻辑。

本系统通过 mybatis-plus-spring-boot3-starter 集成数据访问层，Mapper 层普遍继承 BaseMapper 以提供标准增删改查能力，服务层结合 LambdaQueryWrapper 组织多条件过滤与联动检索，并使用 Page 完成预约、医生、患者等列表的分页查询。实体类使用 @TableName、@TableId 与 @TableField 建立表与字段映射，配合 map-underscore-to-camel-case 配置保证数据库下划线字段与 Java 属性的自动转换；对于诊疗记录、处方与排班等复杂业务场景，系统在 mapper XML 中补充自定义 SQL，以保持查询性能与字段表达的精确性。

2.4 MySQL 数据库

MySQL 8.0 是成熟的关系型数据库管理系统，具备完善的事务支持、索引机制与行级锁控制能力，适合承载高并发下的门诊业务数据读写。本系统采用 InnoDB 引擎与 utf8mb4 编码建立 hospital_db 数据库，通过自增主键、唯一索引与业务索引保障预约编号、医生编号等核心字段的唯一性与查询效率，并结合事务机制确保预约、处方与药品库存等关键数据的修改过程可追溯且一致，满足门诊场景下对数据可靠性与并发稳定性的要求。

2.5 Java 语言

Java 作为跨平台的面向对象语言，具备完备的类库与成熟的生态工具链，适合构建结构清晰、可维护性强的企业级应用。系统后端基于 Java 17 开发，利用类与接口组织业务模型，结合异常处理与泛型集合提升代码的稳健性与复用性；时间处理采用 LocalDate 与 LocalDateTime 保证预约与诊疗记录的时间语义准确。项目中广泛使用 Lombok 的 @Data 等注解简化实体样板代码，并借助 Stream API 完成集合过滤与映射操作，使预约、排班、患者与医生信息的处理逻辑更为简洁一致，为系统长期迭代提供稳定的语言基础。
